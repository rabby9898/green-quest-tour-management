###### 1. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
let greeting;
greetign = {};
console.log(greetign);
```

- A: `{}`
- B: `ReferenceError: greetign is not defined`
- C: `undefined`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: A: `{}`

<i>Because in the code, an empty object is explicitly assigned to the variable greetign using the statement greetign = {}. This means that the variable greetign now holds an empty object, and when you log it to the console using console.log(greetign);, it displays the empty object as the output.</i>

</p>
</details>

```javascript
function sum(a, b) {

sum(1, "2");
```

<p>

<i>because JavaScript interpreter will treat the + operator as a concatenation operator when one of the operands is a string.</i>

</p>

###### 3. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
const food = ["🍕", "🍫", "🥑", "🍔"];
const info = { favoriteFood: food[0] };

info.favoriteFood = "🍝";

console.log(food);
```

- A: `['🍕', '🍫', '🥑', '🍔']`
- B: `['🍝', '🍫', '🥑', '🍔']`
- C: `['🍝', '🍕', '🍫', '🥑', '🍔']`
- D: `ReferenceError`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: A: `['🍕', '🍫', '🥑', '🍔']`

<i>The reason for this output is that the assignment info.favoriteFood = "🍝"; modifies the favoriteFood property of the info object to the string "🍝", but it doesn't affect the food array. Therefore, when you log the food array to the console, it remains unchanged, and its contents are still ['🍕', '🍫', '🥑', '🍔']</i>

</p>
</details>

###### 4. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
function sayHi(name) {
  return `Hi there, ${name}`;
}

console.log(sayHi());
```

- A: `Hi there,`
- B: `Hi there, undefined`
- C: `Hi there, null`
- D: `ReferenceError`

<details><summary><b>Answer</b></summary>
<p>

#### Answer: B: `Hi there, undefined`

<i>The reason for this output is that the sayHi function expects an argument name to be passed when called. In this case, the function is called without any arguments, so name is undefined, resulting in the output "Hi there, undefined"</i>

</p>
</details>

###### 5. Write the `correct answer` from the following options and give an explanation (2-5 lines).

```javascript
let count = 0;
const nums = [0, 1, 2, 3];

nums.forEach((num) => {
  if (num) count += 1;
});

console.log(count);
```

- A: 1
- B: 2
- C: 3
- D: 4

<details><summary><b>Answer</b></summary>
<p>

#### Answer: C: 3

<iThe reason for this output is that the forEach method iterates over the nums array and for each element num, it checks if num is truthy (i.e., not equal to 0). For the elements 1, 2, and 3 in the array, the condition is true, so count is incremented for each of them. As a result, count becomes 3, and that's what's logged to the console.</i>

</p>
</details>
